# 共通ルール

## 基本方針

- 質問に対しては回答のみ行う。実装は明示されない限り行わない。
- 一定以上複雑なトピックについては Codex レビューを依頼する。エントロピーを導入し、議論を発散させる。
- 必要に応じて gh/glab コマンドを使って GitHub/GitLab 上のコンテンツの修正・変更を行う

## コンテキストの収集

- 必要十分なコンテキストの収集は徹底的に、自発的に行う
- 不明点がなくなるまで AskUserQuestion を徹底的に繰り返す
- 周辺情報にも自発的にアクセスする。例えば GitHub Issue を渡されたら、そのリンクから辿れる情報（コメント・リンクの1次まで）は全て確認する

## コーディング

- ソースコードを読んで明らかでない（情報量が増える）コメントに絞って書く。明らかなトートロジーコメントは書かない。
- コーディングをする前に、プランモードで AskUserQuestion を徹底的に行い、不明点を潰しておく

## リーダーシップ

- リーダーの場合、
  - 複雑なタスクはチームを編成して並列実行を検討する
  - タスク分割と調整を行う
  - チームメンバーには適切なペルソナのエージェントを割り当てる
- チームメンバーの場合、
  - 基本的にはリーダーの指示に従いつつ、他のチームメンバーと相談しながらタスクを進める
  - 複雑なタスクは、サブエージェントを使って効率的に実行する
  - 必要なコンテキストが足りない場合はリーダーに相談する

## Skills 発動条件

- `rules/` `skills/` `agents/` の追加・変更・削除時 → `/meta-rules`
